Amazon Connect IVR (Reference)

A reference repository for Amazon Connect architecture, call flows, routing, configuration, and troubleshooting with CloudWatch. Covers inbound/outbound IVR, queues, routing profiles, security, and debugging with contact flow diagrams and docs.

  • Connect Architecture — Instance structure, contact flow engine, and core components.
  • Inbound Call Flow — How inbound calls are received, IVR, and queue transfer.
  • Outbound Call Flow — Outbound campaigns, callbacks, and dialer flows.
  • Match Score Routing — Attribute-based and skill-based routing.
  • Queues & Routing — Queue setup, hours, capacity; agent-queue assignment and channel routing.
  • Security Profiles — Permissions and access control in the Connect UI.
  • CloudWatch Troubleshooting — Call logs, metrics, and debugging failed or unexpected call paths.

Contents: Architecture, inbound/outbound flows, match score routing, queues configuration, routing profiles, security profiles, and CloudWatch troubleshooting. Includes contact flow diagrams (e.g. inbound IVR, attribute-based routing and callback). Prerequisites: Amazon Connect instance; IAM permissions for Connect and CloudWatch.

Clinical Adverse Event Detection AI

A system that helps healthcare teams identify and understand adverse events in clinical data. It combines advanced analytics with clear explanations to improve patient safety and meet compliance requirements. The architecture is modular and easy to extend or integrate with other tools.

  • Adverse Event Detection — Finds potential adverse events in clinical data using modern analytics.
  • Explainability — Gives straightforward reasons and context for each event flagged.
  • Flexible Data Ingestion — Handles both structured and unstructured healthcare data.
  • RAG (Retrieval-Augmented Generation) — Combines data retrieval and generative models for better decision support.
  • Modular Design — Backend split into agent (assessment intake, contract validation, detection API) and ingestion (data pipelines, vector embeddings).

Workflow: Data ingestion → secure storage (e.g. MongoDB) → AI/ML processing for adverse event detection → RESTful API serving with explainable outputs. Built with Python, supports Docker, Azure OpenAI or HuggingFace, and LangSmith for observability.

BugTrackingSystem (BugHound)

A web-based bug recording and tracking application for authorized users. It helps teams manage bugs across multiple products with streamlined communication and resolution. Bug reports are stored in a relational database (PostgreSQL) and accessible via a React frontend and Spring Boot backend.

  • User-friendly Interface — Create, edit, and update bug reports from the browser.
  • Relational Storage — Error reports in SQL tables for easy retrieval and analysis.
  • Search — Find bugs by program, severity, status, and other fields.
  • Dynamic Fields — Release and version data tied to program changes.
  • Attachments — Add images, memory dumps, and text files to bug reports.
  • Secure Access — Login required; role-based access (User, Manager, Admin).

Report fields: Problem summary, report type, severity, reproducibility, suggested fix, assigned to, status, priority, resolution, and more. Stack: React frontend, Spring Boot (Java 17) backend, PostgreSQL. Run locally or with Docker Compose; demo at http://localhost:8080/login.
